# Harness v2 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use `superpowers:subagent-driven-development` to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 게이트가 실질적 의미를 갖도록 harness를 재설계 — 차단보다 인지 강화 우선, 우회 불가, 공통 아키텍처 정립.

**Architecture:** 3-레이어 구조 (Context Injection → Soft Gates → Hard Gates). `hook-common.sh`가 모든 hook의 공통 기반. 신규 SessionStart/Stop hook이 세션 레벨 컨텍스트를 담당.

**Tech Stack:** Bash, Node.js (기존 hook과 동일), jq, git

**Spec:** `docs/superpowers/specs/2026-05-21-harness-v2-design.md`

---

## 의존성 그래프

```
hook-common.sh (공통 라이브러리)
    │
    ├── session-start-context.sh   (SessionStart — 세션 컨텍스트 주입)
    ├── pre-commit-context.sh      (PreToolUse/Bash — 커밋 전 상황 주입)
    ├── persona-inject.sh          (PreToolUse/Edit|Write — 페르소나 주입)
    ├── guard-tdd-write.sh         (PreToolUse/Write — TDD 토큰 게이트)
    ├── guard-tdd-edit.sh          (PreToolUse/Edit — TDD Edit ask)
    ├── guard-commit-message.sh    (PreToolUse/Bash — Conventional Commits)
    ├── guard-code-review.sh       (기존 수정 — bypass 제거)
    ├── guard-static-analysis.sh   (기존 수정 — bypass 제거)
    ├── guard-coverage.sh          (기존 수정 — bypass 제거)
    └── session-stop-report.sh     (Stop — 세션 종료 리포트)
        │
        └── settings.json          (최종 연결)
```

**구현 순서:** hook-common.sh → Layer 1 hooks → Layer 2/3 hooks → 기존 수정 → settings.json

## Task 1: hook-common.sh — 공통 라이브러리

**Description:** 5개 hook에 복사된 중복 로직(git commit 감지, git root, deny/ask JSON, violation 로깅)을 단일 라이브러리로 추출. 이후 모든 hook이 `source`로 사용한다.

**Acceptance criteria:**
- [ ] `is_git_commit "$CMD"` — git commit 명령 감지, 0=커밋 1=아님
- [ ] `git_root` — `git rev-parse --show-toplevel` 래퍼, 실패 시 빈 문자열
- [ ] `deny "메시지"` — `permissionDecision: deny` JSON을 stdout으로 출력 후 exit 0
- [ ] `ask "메시지"` — `permissionDecision: ask` JSON을 stdout으로 출력 후 exit 0
- [ ] `log_violation "type" "file" "detail"` — `~/.pi/hooks/violations.jsonl`에 JSONL 한 줄 추가
- [ ] `log_violation`은 violations.jsonl 디렉터리가 없으면 자동 생성
- [ ] `bash -n hook-common.sh` 문법 오류 없음

**Verification:**
```bash
# 문법 검사
bash -n .pi/hooks/hook-common.sh

# is_git_commit 검사
source .pi/hooks/hook-common.sh
is_git_commit "git -C /repo commit -m 'test'" && echo "PASS" || echo "FAIL"  # → PASS
is_git_commit "git -C /repo status" && echo "FAIL" || echo "PASS"            # → PASS

# deny 출력 검사
OUTPUT=$(echo "" | (source .pi/hooks/hook-common.sh && deny "테스트 메시지"))
echo "$OUTPUT" | jq -e '.hookSpecificOutput.permissionDecision == "deny"'    # → true

# ask 출력 검사
OUTPUT=$(echo "" | (source .pi/hooks/hook-common.sh && ask "테스트"))
echo "$OUTPUT" | jq -e '.hookSpecificOutput.permissionDecision == "ask"'     # → true

# log_violation 검사
source .pi/hooks/hook-common.sh
log_violation "test-type" "TestFile.java" "test detail"
tail -1 ~/.pi/hooks/violations.jsonl | jq -e '.type == "test-type"'      # → true
```

**Dependencies:** None

**Files:**
- Create: `.pi/hooks/hook-common.sh`

**Estimated scope:** S

## Task 4: guard-tdd-write.sh — TDD Write 게이트 (토큰 메커니즘)

**Description:** `guard-test-first.sh`를 대체. C+D 조합: 테스트 파일 Write 시 `@Test` 확인 후 토큰 생성, 구현 파일 Write 시 토큰 확인. 토큰 없으면 deny.

**Acceptance criteria:**
- [ ] Write 대상이 `src/test/java/**/*Test.java`이고 content에 `@Test`가 있으면 `.pi/hooks/gates/tdd-{ClassName}` 토큰 생성 후 통과
- [ ] Write 대상이 `src/test/java/**/*Test.java`이고 content에 `@Test`가 없으면 통과만 (토큰 미생성)
- [ ] Write 대상이 `src/main/java/**/*.java`이고 토큰이 있으면 통과 후 토큰 삭제
- [ ] Write 대상이 `src/main/java/**/*.java`이고 토큰이 없으면 deny (메시지에 필요 파일명 포함)
- [ ] DTO/Record/Config/Properties 제외 패턴 유지 (기존 guard-test-first.sh 로직)
- [ ] `log_violation` 호출: deny 발동 시 `"type":"tdd-write"` 기록
- [ ] `bash -n` 문법 오류 없음

**Verification:**
```bash
bash -n .pi/hooks/guard-tdd-write.sh

# 테스트 파일 Write → 토큰 생성
INPUT='{"tool_name":"Write","tool_input":{"file_path":"/repo/src/test/java/com/example/UserServiceTest.java","content":"@Test\npublic void test() {}"}}'
echo "$INPUT" | bash .pi/hooks/guard-tdd-write.sh
ls .pi/hooks/gates/tdd-UserService 2>/dev/null && echo "PASS: 토큰 생성" || echo "FAIL"

# 구현 파일 Write (토큰 있음) → 통과
INPUT='{"tool_name":"Write","tool_input":{"file_path":"/repo/src/main/java/com/example/UserService.java","content":"public class UserService {}"}}'
OUTPUT=$(echo "$INPUT" | bash .pi/hooks/guard-tdd-write.sh)
[ -z "$OUTPUT" ] && echo "PASS: 통과" || echo "FAIL: 차단됨"
ls .pi/hooks/gates/tdd-UserService 2>/dev/null && echo "FAIL: 토큰 잔류" || echo "PASS: 토큰 삭제"

# 구현 파일 Write (토큰 없음) → deny
INPUT='{"tool_name":"Write","tool_input":{"file_path":"/repo/src/main/java/com/example/OrderService.java","content":"public class OrderService {}"}}'
OUTPUT=$(echo "$INPUT" | bash .pi/hooks/guard-tdd-write.sh)
echo "$OUTPUT" | jq -e '.hookSpecificOutput.permissionDecision == "deny"'    # → true
```

**Dependencies:** Task 1 (hook-common.sh)

## Task 5: guard-tdd-edit.sh — TDD Edit 소프트 게이트

**Description:** 신규. Edit으로 production Java 파일 수정 시, 대응 테스트 파일의 git 변경 여부를 확인하고 변경이 없으면 `ask`로 메인 에이전트에게 판단 위임.

**Acceptance criteria:**
- [ ] Edit 대상이 `src/main/java/**/*.java`가 아니면 exit 0
- [ ] DTO/Record/Config/Properties 패턴이면 exit 0 (테스트 불필요 클래스)
- [ ] 대응 테스트 파일 자체가 없으면 `ask` (테스트 파일 경로 포함)
- [ ] 테스트 파일이 있지만 `git diff HEAD`에 변경 없으면 `ask` (테스트 상태 포함)
- [ ] 테스트 파일이 있고 `git diff HEAD`에 변경 있으면 exit 0 (통과)
- [ ] ask 발동 시 `log_violation` 호출: `"type":"tdd-edit-ask"`
- [ ] `bash -n` 문법 오류 없음

**Verification:**
```bash
bash -n .pi/hooks/guard-tdd-edit.sh

# 테스트 변경 없이 구현 수정 → ask
INPUT='{"tool_name":"Edit","tool_input":{"file_path":"/repo/src/main/java/com/example/UserService.java","old_string":"a","new_string":"b"}}'
OUTPUT=$(echo "$INPUT" | bash .pi/hooks/guard-tdd-edit.sh)

# DTO 파일 → 통과
INPUT='{"tool_name":"Edit","tool_input":{"file_path":"/repo/src/main/java/com/example/UserRequest.java","old_string":"a","new_string":"b"}}'
OUTPUT=$(echo "$INPUT" | bash .pi/hooks/guard-tdd-edit.sh)
[ -z "$OUTPUT" ] && echo "PASS" || echo "FAIL"
```

## Task 7: 기존 3개 hook bypass 제거

**Description:** `guard-coverage.sh`, `guard-code-review.sh`, `guard-static-analysis.sh`에서 env var bypass (`COVERAGE_SKIP=1` 등)를 제거하고, 실패 시 `ask`로 전환. `log_violation` 추가.

**Acceptance criteria:**
- [ ] `guard-coverage.sh`: `COVERAGE_SKIP=1` 체크 로직 제거, 임계값 미달 시 `deny` → `ask` 전환, `log_violation("bypass-ask", ...)` 추가
- [ ] `guard-code-review.sh`: `REVIEW_SKIP=1` 체크 로직 제거, 게이트 미통과 시 deny 유지 (코드리뷰는 soft gate 아님), `log_violation("code-review-deny", ...)` 추가
- [ ] `guard-static-analysis.sh`: `STATIC_ANALYSIS_SKIP=1` 체크 로직 제거, `log_violation("static-analysis-deny", ...)` 추가
- [ ] 세 hook 모두 `source hook-common.sh` 사용으로 공통 함수 교체
- [ ] 기존 `deny()` 함수 인라인 구현 → `hook-common.sh`의 `deny` 사용
- [ ] `bash -n` 문법 오류 없음

**Verification:**
```bash
# coverage: env var 무시 확인
COVERAGE_SKIP=1 bash -c 'echo "{}" | bash .pi/hooks/guard-coverage.sh'
# → env var 설정해도 동일하게 동작해야 함 (bypass 없어야 함)

# code-review: REVIEW_SKIP=1 무시 확인

echo "$OUTPUT" | jq -e '.hookSpecificOutput.permissionDecision == "deny"'    # → true

# static analysis: 문법 검사
bash -n .pi/hooks/guard-static-analysis.sh
```

## Task 8: guard-commit-message.sh — Conventional Commits 게이트

**Description:** 신규. git commit 시 커밋 메시지가 Conventional Commits 형식인지 검사. 형식 오류 시 deny.

**Acceptance criteria:**
- [ ] git commit이 아닌 명령이면 exit 0
- [ ] `-m` 플래그 또는 heredoc(`EOF`) 방식의 메시지 추출
- [ ] 정규식: `^(feat|fix|chore|refactor|docs|test|perf|ci|style|revert)(\([a-z0-9][a-z0-9-]*\))?: .{1,100}$`
- [ ] 형식 맞으면 exit 0
- [ ] 형식 오류 시 deny (현재 메시지와 올바른 예시 포함)
- [ ] `--amend` 커밋도 동일하게 검사
- [ ] `log_violation("commit-message", ...)` 추가
- [ ] `bash -n` 문법 오류 없음

**Verification:**
```bash
bash -n .pi/hooks/guard-commit-message.sh

# 올바른 형식 → 통과
INPUT='{"tool_name":"Bash","tool_input":{"command":"git commit -m \"feat(user): 사용자 생성 API 추가\""}}'
OUTPUT=$(echo "$INPUT" | bash .pi/hooks/guard-commit-message.sh)
[ -z "$OUTPUT" ] && echo "PASS" || echo "FAIL"

# 틀린 형식 → deny
INPUT='{"tool_name":"Bash","tool_input":{"command":"git commit -m \"update service\""}}'
OUTPUT=$(echo "$INPUT" | bash .pi/hooks/guard-commit-message.sh)
echo "$OUTPUT" | jq -e '.hookSpecificOutput.permissionDecision == "deny"'    # → true

# 비커밋 → 통과
INPUT='{"tool_name":"Bash","tool_input":{"command":"git push origin feat/xxx"}}'
OUTPUT=$(echo "$INPUT" | bash .pi/hooks/guard-commit-message.sh)
[ -z "$OUTPUT" ] && echo "PASS" || echo "FAIL"
```

**Dependencies:** Task 1 (hook-common.sh)

## Task 9: session-stop-report.sh — 세션 종료 리포트

**Description:** Stop hook. 세션 종료 시 현재 세션 동안 발생한 violation 요약을 출력. violations.jsonl에서 오늘 날짜 기준 필터.

**Acceptance criteria:**
- [ ] Stop 이벤트 수신 시 `additionalContext` JSON 출력
- [ ] 오늘 날짜(`date +%Y-%m-%d`) 기준 violations.jsonl 필터
- [ ] type별 발동 횟수 집계 출력
- [ ] violations.jsonl이 없거나 오늘 기록이 없으면 "이번 세션 게이트 발동 없음" 출력
- [ ] jq 없이도 동작 (jq 없으면 raw 출력으로 fallback)
- [ ] `bash -n` 문법 오류 없음

**Verification:**
```bash
bash -n .pi/hooks/session-stop-report.sh
```
